# redis

## 基本数据类型

String\List\Set\SortSet\Hash  
SortSet底层由跳表实现(数量小于512且数据为数字时为有序数组)

## 应用

布隆过滤器：过滤无效请求  
分布式锁：setnx key value time  
缓存：set key value time  
排行榜：zset key value

## redis为什么快

1.内存操作  
2.单线程，无需切换上下文  
3.非阻塞I/O多路复用，单线程监听多个套接字和已连接套接字，不会阻塞在单个请求上，而是一直在真正的处理数据（类似的如netty框架）  
4.高效的数据结构，如哈希表和跳表

## 数据持久化

1.AOF：命令执行结束后，记录日志到磁盘，三种写回策略Always、Everysec、No（可靠性从高到低，性能从低到高）。  
定时会进行AOF重写，避免一直追加日志过大  
2.RDB：内存快照，但相对耗时耗力，频率过快可能影响正常读写。  
数据不能丢失时，内存快照和 AOF 的混合使用是一个很好的选择；  
如果允许分钟级别的数据丢失，可以只使用 RDB；  
如果只用 AOF，优先使用 everysec 的配置选项，因为它在可靠性和性能之间取了一个平衡。

## 主从

一主多从，读写分离，仅主库可进行修改，从库只能读取。  
主从首次同步，以来RDB快照获取所有数据并加载，RDB快照之后的操作会记录在内存中发给从库

## 哨兵

1.监控主库状态，判断是否下限  
2.主库下线时，选择新主库  
3.选出新主库，通知其他从库
