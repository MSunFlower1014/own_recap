# JAVA基础

## 集合

HashMap：
默认容量为16，数组加链表实现，哈希值取余获取数组下标，重复时节点变为链表，链表容量大于8时变为红黑树.  
比较时先判空，然后比较哈希值，最后通过equals比较，所以key必须重写hashCode方法。

ArrayList:  
数组列表默认容量为10

## 线程池

new ThreadPoolExecutor(10,10,200L, TimeUnit.MILLISECONDS
,new LinkedBlockingQueue<Runnable>(10),
Executors.defaultThreadFactory(),
new ThreadPoolExecutor.AbortPolicy()   
参数：核心线程数，最大线程数，非核心线程存活时间，存活时间单位，阻塞等待队列，线程工厂，丢弃策略  
执行逻辑：核心线程空闲直接执行，核心满了优先放到阻塞队列等待，阻塞队列满了创建新线程直到达到最大线程数，达到最大线程且阻塞队列满了执行丢弃策略  
ThreadPoolExecutor自带丢弃类型：  
DiscardOldestPolicy-随机丢弃一个老的等待任务  
DiscardPolicy-直接丢弃  
AbortPolicy-抛异常  
CallerRunsPolicy-由提交任务的调用者线程直接执行任务

## 并发类  

ConcurrentHashMap:  
并发安全的hashMap  
CopyOnWriteArrayList:  
并发安全的list，适合读多写少的场景  
CountDownLatch:  
并发的计数器，countDown达到次数后返回false  
AtomicInteger/AtomicLong:  
并发安全的int/long

## 锁

ReentrantLock:  
可重入锁，支持参数指定公平锁还是非公平锁。默认为非公平锁  
synchronized:  
关键字，官方实现的锁，通过在每个对象头维护一个锁来实现。作用在静态类或者对象时锁类，在普通对象时锁对象。  
死锁，互斥条件-请求保持-不可剥夺-循环等待，通过超时放弃、一次分配所有资源来避免。

## 其他

ThreadLocal:  
线程内部的存储类，可以用来存储通用上下文信息，如登录信息、请求来源ip等。很多rpc框架通过此类传递通用的信息。  
实例被回收后，由于时弱引用，key会被回收，但value无法自动回收，线程在不断复用的情况下，value值会越积累越多最终oom。  
线程执行结束需要执行remove方法来清除该变量，避免内存泄露

## 设计模式

工厂模式：如数据库线程连接池  
单例模式：  
建造者模式：Bean的各种builder  
代理模式：aop实现  
门面模式：slog4j的日志实现，不需要关注底层使用什么日志框架，直接使用即可  
观察者模式：订阅通知功能

## mybatis

用#表示字符串，用来防治sql注入。用$表示直接输入，如指定分表表名。  
防治sql注入：参数校验、使用#传入参数、避免向用户直接展示数据库错误信息

## Spring

FactoryBean和BeanFactory：前者为静态工厂或实例工厂，后者是一个特殊接口，用于复杂bean创建如数据库连接池
Spring AOP实现：  
接口类通过动态代理，无接口类通过cglib生成及橙子类（无法代理final、private和static方法）
AspectJ：静态织入，功能比aop更强大，可以在编译期织入、编译后织入、JVM类加载时织入
