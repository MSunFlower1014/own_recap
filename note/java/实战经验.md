# 实战

## 秒杀系统

策略： 层层削峰、尽量走缓存、流量分流。
1.前端： 按钮置灰、CDN 静态化。  
2.网关： 令牌桶限流、黑名单。  
3.缓存： Redis 预热库存、Lua 脚本保证扣减原子性。  
4.异步： MQ 异步下单，提示“排队中”。   
5.数据库： 乐观锁更新，防止超卖。  
6.域名与服务器隔离，避免带崩常规应用。

## cpu飙升100%

top命令查看cpu占用pid；jstack查看堆栈信息（死循环代码）

# 内存泄露

1.jstat -gcutil [PID] 1000 10 （每秒输出一次，关注 O (Old Gen) 和 FGC (Full GC) 次数）。查看老年代内存占用没有明显下降
2.jmap -dump:format=b,file=heap.hprof [PID] 导出堆栈信息。  
3.分析内存占用，使用MAT (Eclipse Memory Analyzer) 或 JProfiler。
4.常见问题：未关闭的资源io泄露；ThreadLocal 没调用 remove()；Metaspace 泄露：大量动态生成类
