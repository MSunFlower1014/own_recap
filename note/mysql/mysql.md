# mysql

## 日志

日志分为binlog和redolog
binlog是server层日志，记录逻辑如某行数据a➕2，也是从库用来同步数据的基础
redolog是引擎层，为innoddb专有，记录物理信息，如a的b字段改为2
undolog回滚日志

避免长事务

## 数据的四特性

原子性 保证单次操作不受干扰
隔离性 多事务冲突处理，分为读未提交，读提交，可重复读，串行化四个级别，oracle默认为读提交，mysql innodb为可重复读
统一性
持久性 数据落盘，binlog

## 索引结构

索引 b➕树，只有叶子结点保存所有数据，其中主键索引叶子保存整行数据，非主键保存索引与主键id，叶子结点相连
优势：树层低，减少搜索次数，每个节点保存更多信息，减少磁盘寻址且一次获取多数据，叶子结点便于范围查询。
主键最好有序递增，避免数据页反复合并分裂，最好用自增主键

## 回表

通过非主键索引查询符合条件主键id后在通过id查询整行数据叫回表。
索引覆盖，如联合索引a，b 仅通过a查询对应的a和b值，不需要回表
最左前缀 联合索引按照字段顺序排序由左到右

## 语句类型

DDL 增删改查行数据
DML 表锁，alter table add/drop

行锁在事务提交时释放
